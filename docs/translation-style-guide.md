# 翻译风格和术语表指南

本文档说明影视配音翻译阶段如何控制对白风格、术语一致性和 TTS 语气提示。

## 核心目标

- 译文要适合中文配音，不追求逐字直译。
- 尽量贴近原始片段时长，避免明显超长或过短。
- 保留必要语气词、停顿、口癖和人物情绪。
- 统一角色称呼、专有名词和关键梗。
- 翻译 provider 建议返回 `target_text`、`emotion`、`style_prompt` 三个字段。

## 剧集类型建议

美剧对白：

- 口语、节奏快，少用书面表达。
- 可以适度本地化固定说法，但不要改写剧情信息。
- 对争吵、讽刺、玩笑等情绪要在 `style_prompt` 中说明。

日剧对白：

- 注意敬语、称呼、上下级关系和停顿。
- 语气词不要机械删除，例如犹豫、请求、惊讶类短音可以保留为自然中文表达。
- 对称呼如“先輩”“先生”“お母さん”优先使用项目术语表。

韩剧对白：

- 注意亲属称谓、前后辈称呼和情绪递进。
- 短句要自然，避免把韩语句式硬搬到中文。
- 强情绪片段建议输出明确 `emotion`，便于后续 TTS 控制。

## 术语表格式

术语表使用 JSON 对象：

```json
{
  "先輩": "前辈",
  "오빠": "哥哥",
  "District 12": "第十二区"
}
```

项目向导可以选择该 JSON 文件；创建项目后会写入 `settings.json`，翻译 prompt 会包含 `源词 -> 译词` 格式的术语约束。

## 输出 Schema

推荐 provider 返回 JSON：

```json
{
  "target_text": "前辈，拜托了。",
  "emotion": "pleading",
  "style_prompt": "带一点请求感，语速自然，保留短暂停顿"
}
```

如果 provider 只能返回字符串，HTTP profile 仍可将其映射为 `target_text`；`emotion` 和 `style_prompt` 可以缺省。

## TTS 衔接

`style_prompt` 会保存到时间线片段，并在单句重生成或整段 TTS 时传给 TTS adapter。建议把说话状态写得短而可执行，例如：

- `克制、低声、语速偏慢`
- `惊讶但不要夸张，句尾上扬`
- `请求感，保留短暂停顿`
