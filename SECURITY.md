# 安全政策

## 报告安全问题

如果你发现安全问题，请先不要公开披露漏洞细节。可以通过 GitHub 的私有安全报告功能报告；如果仓库页面暂未开启该入口，请先创建一个不包含敏感细节的 issue，说明需要私下沟通安全问题。

请在报告中尽量包含：

- 受影响的版本或提交。
- 复现步骤。
- 可能影响的数据、文件或凭据范围。
- 你建议的缓解方式。

## 敏感信息

请不要提交：

- 真实 API key、token、cookie、私钥或账号凭据。
- 未授权的视频、音频、字幕或模型权重。
- 包含个人身份信息的样例素材。

项目示例统一使用 `YOUR_API_KEY`、`{{ api_key }}` 或测试用的 `test-token` 占位符。

## 模型与素材

如果安全问题涉及模型权重、真实视频、真实音频或生成结果，请只描述最小复现信息，不要把原始素材上传到公开 issue。可以使用合成样片、dry-run profile 或脱敏日志复现问题。

## 支持版本

当前项目仍处于早期 v0.x 阶段，安全修复优先落到 `main` 分支。
